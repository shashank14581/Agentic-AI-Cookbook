OVERVIEW

What this pattern teaches

Pattern 19 separates the MCP client from the MCP server. Pattern 18 bundled everything into one notebook cell. This pattern focuses only on the consuming side: given a running MCP endpoint, how does a Python client connect and call a tool?

The mental model is simple: URL → streamable HTTP connection → client session → tool call → text result. This is the core building block for agents that rely on external MCP servers instead of local functions.

import nest_asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

nest_asyncio.apply()


MCP_URL = "http://127.0.0.1:8011/mcp"


class MCPClient:
    def __init__(self, url):
        self.url = url

    async def call_tool(self, tool_name, arguments):
        async with streamablehttp_client(self.url) as (read, write, _):
            async with ClientSession(read, write) as session:
                await session.initialize()

                result = await session.call_tool(
                    name=tool_name,
                    arguments=arguments
                )

                return result.content[0].text


async def main():
    client = MCPClient(MCP_URL)

    result = await client.call_tool(
        tool_name="search_private_knowledge",
        arguments={
            "query": "What is Blue Mango?"
        }
    )

    print(result)


await main()
Mental Model

Line 1: Installs the MCP package and nest_asyncio. This is notebook-friendly setup for running async client code.

Lines 3-7: Imports MCP client utilities and patches the notebook event loop so await main() can run cleanly.

Line 10: Points to the MCP server endpoint. This assumes the Pattern 18 server is already running on port 8011.

Lines 13-27: MCPClient opens a streamable HTTP connection, creates a ClientSession, initializes it, calls the requested tool, and returns the tool text.

Lines 30-41: main() creates the client and calls search_private_knowledge with a query about Blue Mango.

Line 44: Runs the async workflow directly in the notebook. The expected output is the private knowledge response returned by the MCP server.
