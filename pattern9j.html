Pattern 9j – Connecting to an MCP Server

This simple client shows how to call a tool on an existing MCP server. It does not run Gemini; it just demonstrates the protocol itself.

Connecting to a Server

The MCP client needs the URL of a running server. In this example, the server is assumed to be listening on http://127.0.0.1:8011/mcp. Nest asyncio is applied so that asynchronous operations can run inside interactive notebooks.

The Client Wrapper

MCPClient provides a single method, call_tool(), which opens a streaming HTTP connection, initialises a ClientSession, and calls the remote tool by name. It unpacks the returned text content and passes it back to the caller.

Example Invocation

The main() function creates a client, then calls the search_private_knowledge tool with a simple question. The server’s response (for example, a definition of “Blue Mango”) is printed. You can adapt this pattern to call any tool your MCP server exposes.
