GUIDE

How to Read the Scripts

Identify the Agent Classes

Look for BaseAgent or specialized agents that wrap the Gemini API.

Read the Orchestrator

Find functions like orchestrate() or run_conversation().

Trace the Memory

Check how conversation history, facts, and summaries are added to prompts.

Study Tool Schemas

Understand what functions the agent can call and what arguments are expected.

Check Return Types

Notice whether the model returns plain text, JSON, tool calls, or routed control.

PATTERNS

Different View of Each Pattern

1
What it does

Basic Conversational Agents

Creates two or more agents that exchange messages through a simple orchestrator.

How to think about it

This is the foundation. An agent is just a loop: memory goes in, prompt is created, Gemini responds, and the response is stored back into memory.

2
What it does

Personality and Memory

Adds character, identity, facts, and remembered information to the agent.

How to think about it

The agent stops being stateless. It now carries a personality and remembers useful facts, making responses more consistent across turns.

3
What it does

Moderator-Based Speaking

A moderator decides which agent should speak next instead of hardcoding the order.

How to think about it

This is dynamic routing. The conversation is no longer linear; the LLM chooses the next participant based on context.

4
What it does

Agents Using Tools

The agent calls external tools, such as a weather function, when needed.

How to think about it

The model does not need to know everything. It can delegate factual or procedural work to functions and then explain the result.

5
What it does

Agents Returning JSON

Forces the agent to return structured machine-readable output.

How to think about it

This turns language into control data. Instead of only replying, the model returns fields your program can parse and use.

6
What it does

Reasoning Agents and Hand-Off

The agent thinks, responds, and decides which agent should receive control next.

How to think about it

The agent becomes a planner. It not only says something, but also decides the next step in the workflow.

7
What it does

Supervisor Pattern

A supervisor assigns tasks to specialist agents and combines their outputs.

How to think about it

This is like a manager-agent. It does not do every task itself; it routes planning, analysis, writing, or review to the right worker.

8
What it does

Parallel Agents

Multiple agents work at the same time and their outputs are merged later.

How to think about it

This is fan-out and fan-in. Split the problem, let agents work independently, then synthesize the results.

9
What it does

Debate Pattern

Agents argue different sides and a judge evaluates the final answer.

How to think about it

This pattern improves reasoning by forcing disagreement. The judge benefits from seeing competing arguments.

9a
What it does

Long-Term Memory

The agent stores useful insights in a database for future use.

How to think about it

This moves memory from temporary chat history to persistent storage. The agent can learn across sessions.

9b
What it does

Shared Memory

Multiple agents read and write to the same memory store.

How to think about it

This creates a shared workspace. Agents do not operate in isolation; they collaborate through common context.

9c
What it does

Knowledge Extraction

The agent extracts people, companies, locations, products, and other entities.

How to think about it

This converts raw text into structured knowledge. It is the beginning of knowledge graphs and entity memory.

9d
What it does

Chunking and Embedding

Documents are split into chunks and converted into vectors.

How to think about it

This prepares documents for semantic search. The agent cannot retrieve meaning until content has been embedded.

9e
What it does

Vector Search Before Generation

The agent searches relevant chunks before producing an answer.

How to think about it

This grounds the output. The agent answers using retrieved evidence instead of relying only on model memory.

9f
What it does

Basic RAG Agent

Combines retrieval with generation to answer questions using external context.

How to think about it

RAG is open-book answering. The model reads relevant notes first, then generates an answer.

9g
What it does

Agentic RAG

The agent decides whether retrieval is needed before answering.

How to think about it

This makes RAG smarter. Retrieval becomes conditional, not automatic.

9h
What it does

Multi-Hop Chain Retrieval

The agent retrieves information across multiple connected steps.

How to think about it

Some answers require multiple searches. The agent retrieves, reasons, then retrieves again.

9i
What it does

MCP Protocol Walkthrough

Introduces the protocol that lets agents connect with external tools and context servers.

How to think about it

MCP is a standard interface. Instead of hardcoding every tool, the agent talks to a server that exposes tools.

9j
What it does

Connecting to an MCP Server

Shows how a client connects to a running MCP server.

How to think about it

The agent becomes a client. It discovers tools from the server instead of owning them directly.

9k
What it does

Creating Your Own Tool Server

Builds a custom server that exposes your own tools to agents.

How to think about it

This separates intelligence from capability. The model reasons, while the server provides actions.

9l
What it does

Agent + MCP Full Integration

Combines an agent, MCP client, MCP server, and external tools into one complete system.

How to think about it

This is the full architecture. The agent reasons, discovers tools, calls the server, receives results, and generates the final answer.

Final Thoughts

This repository moves from basic agent conversations to serious agentic architecture: memory, tools, supervision, retrieval, MCP, and full integration.

Read the scripts in order. Each one adds a new capability to the previous mental model. By the end, you are no longer just prompting a model — you are designing an intelligent system.
