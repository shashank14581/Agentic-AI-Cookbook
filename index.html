Getting a Gemini API Key

To run any of these agents, you need an API key for Google Gemini. Follow these steps to create one in Google AI Studio.

1. Sign in to AI Studio

Go to Google AI Studio and log in with your Google account. AI Studio is where you manage Gemini API access and projects.

2. Open the API Keys Page

Once inside, select API Keys from the left navigation panel.

3. Create an API Key

Click Get API Key. Select an existing Google Cloud project or create a new one, then click Create API Key.

4. Copy the Key

Copy your Gemini API key and store it securely. You will need it in your environment variables or configuration files.

How to Read the Scripts

1. Identify the Agent Classes

Most scripts define a BaseAgent or specialized agent classes that wrap the Gemini API. The simplest versions maintain conversation history and send prompts to Gemini.

2. Read the Orchestration Logic

Functions such as orchestrate() or run_conversation() coordinate which agent speaks next and handle message routing between agents.

3. Observe Memory and Prompt Construction

  • Recent conversation summaries
  • Extracted facts
  • System prompts
  • Role-based formatting

4. Examine Tool Definitions

In scripts that use external tools or functions, the tools are declared using JSON schemas and passed to Gemini during prompt construction.

5. Check Return Types

Some agents force Gemini to return structured JSON instead of free-form text. This is useful for reliable workflows and deterministic orchestration.

Mental Models for Each Pattern

Pattern 1

Basic Conversational Agents

A BaseAgent wraps the Gemini API and maintains conversation history. Two agents exchange messages through an orchestrate() function.

  • Build a prompt from memory
  • Send it to Gemini
  • Receive a response
Pattern 2

Personality and Memory

This pattern stores facts about each character and uses both short-term and long-term memory during generation.

  • Short-term memory → recent context
  • Long-term memory → persistent facts
Pattern 3

Moderated Conversations

A Moderator Agent decides who should speak next by inspecting the latest message and returning the next speaker.

  • Conditional routing
  • Dynamic orchestration
  • LLM-directed control flow
Pattern 4

Tools and Structured Outputs

Agents can call external functions using JSON schemas. Gemini chooses the tool, generates arguments, and receives the result back into context.

  1. Define tools
  2. Expose tools to the model
  3. Let the model choose tools
  4. Execute them externally
  5. Return results back to the model
Pattern 5

Reasoning and Hand-Off

The reasoning agent returns structured JSON with a message and the next agent to hand off to.

{
  "message": "...",
  "handoff_to": "ResearchAgent"
}
  • Reasoning → handled by Gemini
  • Control flow → handled by a simple router
Pattern 6

Supervisor Pattern

A Supervisor Agent coordinates specialized workers such as a Planner, Analyst, and Copywriter.

  1. Planning
  2. Analysis
  3. Writing
  4. Integration
Pattern 7

Parallel and Debate Patterns

Parallel agents work independently and then synthesize results. Debate agents argue opposing views while a judge evaluates them.

  • Parallel Pattern → fan-out / fan-in reasoning
  • Debate Pattern → adversarial reasoning
Pattern 8

Long-Term and Shared Memory

Long-term memory stores insights in a database, while shared memory allows multiple agents to read and write to the same store.

  • Database memory → persistence across sessions
  • Shared memory → collaborative multi-agent cognition
Pattern 9

Knowledge Extraction

A specialized agent extracts entities such as people, companies, locations, and products from unstructured text.

  • Knowledge graphs
  • Entity memory
  • Semantic indexing systems
Pattern 10

Chunking, Embedding, and Retrieval

This pattern chunks documents, generates embeddings, performs vector search, and decides when retrieval is necessary.

  1. Decide if retrieval is needed
  2. Retrieve only when useful
  3. Inject retrieved context into generation

Final Thoughts

These mental models are designed to help you understand the architectural decisions behind each script. As you explore the repository, connect the implementation details to the higher-level design patterns described here.

With a Gemini API key and these mental models, you will be well on your way to building your own agentic AI systems.
