Pattern 4: Using Tools

This pattern shows how an agent can call external functions (tools) to fetch data or perform calculations.

Summary

The script defines two real Python functions—one to fetch weather from an external API and another to add numbers. A tool schema exposes these functions to the Gemini model. The Agent class uses the Gemini API’s tool‑calling capabilities: if the model returns a function call, the agent executes the Python function and then sends the result back to Gemini before generating the final answer. The example conversation asks the agents for the weather and demonstrates the tool invocation.


import os
import requests
from google import genai
from google.genai import types

# -------- REAL TOOL --------
def get_weather(city):
    geo = requests.get(
        f"https://geocoding-api.open-meteo.com/v1/search?name={city}"
    ).json()
    if "results" not in geo:
        return f"{city} not found"
    lat = geo["results"][0]["latitude"]
    lon = geo["results"][0]["longitude"]
    weather = requests.get(
        f"https://api.open-meteo.com/v1/forecast?"
        f"latitude={lat}&longitude={lon}¤t_weather=true"
    ).json()
    data = weather.get("current_weather", {})
    return f"{city}: {data.get('temperature')}°C"

def add_numbers(a, b):
    return a + b

# -------- TOOL SCHEMA --------
tools = [
    types.Tool(
        function_declarations=[
            types.FunctionDeclaration(
                name="get_weather",
                description="Get real weather",
                parameters={
                    "type": "OBJECT",
                    "properties": {"city": {"type": "STRING"}},
                    "required": ["city"]
                }
            ),
            types.FunctionDeclaration(
                name="add_numbers",
                description="Add numbers",
                parameters={
                    "type": "OBJECT",
                    "properties": {
                        "a": {"type": "NUMBER"},
                        "b": {"type": "NUMBER"}
                    },
                    "required": ["a", "b"]
                }
            )
        ]
    )
]

# -------- AGENT --------
class Agent:
    def __init__(self, name, prompt):
        self.name = name
        self.prompt = prompt
        self.client = genai.Client(api_key=os.environ.get("GEMINI_API_KEY"))
        self.model = "gemini-2.5-flash"

    def think(self, text):
        if not text:
            text = "continue"
        contents = [
            types.Content(
                role="user",
                parts=[types.Part.from_text(text=text)]
            )
        ]
        config = types.GenerateContentConfig(
            system_instruction=[types.Part.from_text(text=self.prompt)],
            tools=tools
        )
        response = self.client.models.generate_content(
            model=self.model,
            contents=contents,
            config=config
        )
        # -------- CHECK TOOL --------
        fc = None
        for p in response.candidates[0].content.parts:
            if p.function_call:
                fc = p.function_call
                break
        # -------- IF TOOL --------
        if fc:
            name, args = fc.name, fc.args
            print(f"\n⚙️ {self.name} -> {name} {args}")
            if name == "get_weather":
                result = get_weather(**args)
            elif name == "add_numbers":
                result = add_numbers(**args)
            else:
                result = "Unknown tool"
            print(f"🔧 {result}")
            tool_response = types.Content(
                role="tool",
                parts=[
                    types.Part.from_function_response(
                        name=name,
                        response={"result": result}
                    )
                ]
            )
            final = self.client.models.generate_content(
                model=self.model,
                contents=[
                    contents[0],
                    types.Content(role="model", parts=response.candidates[0].content.parts),
                    tool_response
                ],
                config=config
            )
            # -------- EXTRACT TEXT --------
            if final.text:
                output = final.text
            else:
                output = ""
                for p in final.candidates[0].content.parts:
                    if p.text:
                        output += p.text
        else:
            if response.text:
                output = response.text
            else:
                output = ""
                for p in response.candidates[0].content.parts:
                    if p.text:
                        output += p.text
        print(output)
        return output

# -------- RUN --------
if __name__ == "__main__":
    human = Agent("Shashank", "You are a human")
    dog = Agent("Bruno", "You are a smart dog")
    msg = "What's the weather in London?"
    for _ in range(1):
        print(f"\nShashank:", end=" ")
        msg = human.think(msg)
        print(f"\nBruno:", end=" ")
        msg = dog.think(msg)

Lines 1–3: Import os, requests, and the Gemini client. requests is used to call external APIs.

Lines 6–25: Define get_weather() which uses the Open‑Meteo geocoding and weather APIs to get the current temperature for a city. If the city isn’t found, it returns an error message. add_numbers() simply returns the sum of two numbers.

Lines 32–60: Create a tools list containing a single Tool object. Each function is declared with a name, description and JSON schema describing its arguments. These declarations allow Gemini to call the Python functions when appropriate.

Lines 63–161: Define the Agent class. The think() method builds the request with a system instruction and the tools. When Gemini returns, it inspects the output for a function_call. If a function call exists, it executes the corresponding Python function and sends the result back to the model. Finally, it extracts the final message text and returns it.

Lines 164–178: In the main block, create a human and a dog agent. The conversation asks for the weather in London, demonstrating how the model calls the get_weather tool, prints the intermediate result (with the ⚙️ and 🔧 icons), and then integrates the tool response into its final output.
