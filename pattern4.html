Pattern 4 · Tools

Agents Using Tools

In this pattern, agents move beyond pure conversation. They can ask Gemini to select external Python functions, execute them, and feed the result back into the model before producing a final answer.

The agent gets hands.

Tools let an agent interact with the outside world instead of only generating text.

Gemini selects a tool
Python executes the function
Tool result returns to Gemini

OVERVIEW

What this pattern teaches

The script defines two real Python functions: get_weather(), which fetches weather from an external API, and add_numbers(), which performs a simple calculation.

A tool schema exposes these functions to Gemini. If Gemini returns a function call, the agent executes the Python function, sends the result back to Gemini, and then generates the final response.

CODE

Line-by-line walkthrough

pattern4.py 
import os
import requests
from google import genai
from google.genai import types

# Real tool
def get_weather(city):

    geo = requests.get(
        f"https://geocoding-api.open-meteo.com/v1/search?name={city}"
    ).json()

    if "results" not in geo:
        return f"{city} not found"

    lat = geo["results"][0]["latitude"]
    lon = geo["results"][0]["longitude"]

    weather = requests.get(
        f"https://api.open-meteo.com/v1/forecast?"
        f"latitude={lat}&longitude={lon}¤t_weather=true"
    ).json()

    data = weather.get("current_weather", {})

    return f"{city}: {data.get('temperature')}°C"

def add_numbers(a, b):

    return a + b

# Tool schema
tools = [

    types.Tool(

        function_declarations=[

            types.FunctionDeclaration(
                name="get_weather",
                description="Get real weather",
                parameters={
                    "type": "OBJECT",
                    "properties": {"city": {"type": "STRING"}},
                    "required": ["city"]
                }
            ),

            types.FunctionDeclaration(
                name="add_numbers",
                description="Add numbers",
                parameters={
                    "type": "OBJECT",
                    "properties": {
                        "a": {"type": "NUMBER"},
                        "b": {"type": "NUMBER"}
                    },
                    "required": ["a", "b"]
                }
            )

        ]

    )

]

# Agent
class Agent:

    def __init__(self, name, prompt):

        self.name = name
        self.prompt = prompt
        self.client = genai.Client(
            api_key=os.environ.get("GEMINI_API_KEY")
        )
        self.model = "gemini-2.5-flash"

    def think(self, text):

        if not text:
            text = "continue"

        contents = [
            types.Content(
                role="user",
                parts=[types.Part.from_text(text=text)]
            )
        ]

        config = types.GenerateContentConfig(
            system_instruction=[types.Part.from_text(text=self.prompt)],
            tools=tools
        )

        response = self.client.models.generate_content(
            model=self.model,
            contents=contents,
            config=config
        )

        # Check tool
        fc = None

        for p in response.candidates[0].content.parts:

            if p.function_call:

                fc = p.function_call

                break

        # If tool
        if fc:

            name, args = fc.name, fc.args

            print(f"\\n⚙️ {self.name} -> {name} {args}")

            if name == "get_weather":
                result = get_weather(**args)

            elif name == "add_numbers":
                result = add_numbers(**args)

            else:
                result = "Unknown tool"

            print(f"🔧 {result}")

            tool_response = types.Content(
                role="tool",
                parts=[
                    types.Part.from_function_response(
                        name=name,
                        response={"result": result}
                    )
                ]
            )

            final = self.client.models.generate_content(
                model=self.model,
                contents=[
                    contents[0],
                    types.Content(
                        role="model",
                        parts=response.candidates[0].content.parts
                    ),
                    tool_response
                ],
                config=config
            )

            if final.text:
                output = final.text

            else:

                output = ""

                for p in final.candidates[0].content.parts:

                    if p.text:
                        output += p.text

        else:

            if response.text:
                output = response.text

            else:

                output = ""

                for p in response.candidates[0].content.parts:

                    if p.text:
                        output += p.text

        print(output)

        return output

# Run
if __name__ == "__main__":

    human = Agent("Shashank", "You are a human")
    dog = Agent("Bruno", "You are a smart dog")

    msg = "What's the weather in London?"

    for _ in range(1):

        print(f"\\nShashank:", end=" ")
        msg = human.think(msg)

        print(f"\\nBruno:", end=" ")
        msg = dog.think(msg)
Mental Model

Lines 1–4: Import os, requests, and the Gemini SDK. requests lets the tool call external APIs.

Lines 7–28: Define get_weather(). It first calls a geocoding API to convert a city into latitude and longitude, then calls the weather API to get current temperature.

Lines 30–32: Define add_numbers(), a tiny calculation tool. It demonstrates that tools can be both API calls and local functions.

Lines 35–76: Create the tool schema. This tells Gemini which tools exist, what they do, and what arguments they require.

Lines 79–99: Define the Agent class and initialize the Gemini client. The agent receives both a name and a system prompt.

Lines 101–121: Build the user message and generation config. The key detail is that tools=tools exposes the Python functions to Gemini.

Lines 123–139: Inspect Gemini’s response to check whether it requested a function call. If a function call exists, the agent captures its name and arguments.

Lines 142–181: Execute the selected Python function. Then build a tool response and send it back to Gemini so the model can produce a natural final answer.

Lines 183–207: Extract the final text response whether Gemini used a tool or answered directly.

Lines 212–223: Create a human agent and a dog agent, then ask about London weather. This demonstrates real tool invocation inside an agent conversation.
