OVERVIEW

What this pattern teaches

The Long-Term Memory Agent (Pattern 10) demonstrates how to bridge external data sources (RSS feeds) with persistent storage (SQLite). The agent evaluates news articles not just in isolation, but by checking its existing knowledge base first.

This pattern moves beyond simple RAG (Retrieval Augmented Generation) by giving the agent Agency over its own memory—deciding what is "meaningful" to save and what to ignore, building a custom intelligence layer.

import os
import sqlite3
import requests
import feedparser
from bs4 import BeautifulSoup
from google import genai
from google.genai import types

def init_db():
    conn = sqlite3.connect("agent_memory.db")
    cursor = conn.cursor()
    cursor.execute("""CREATE TABLE IF NOT EXISTS memory (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        content TEXT, source TEXT, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP)""")
    conn.commit()
    conn.close()

def save_to_db(content, source):
    conn = sqlite3.connect("agent_memory.db")
    cursor = conn.cursor()
    cursor.execute("INSERT INTO memory (content, source) VALUES (?, ?)", (content, source))
    conn.commit()
    conn.close()

def load_memory(limit=5):
    conn = sqlite3.connect("agent_memory.db")
    cursor = conn.cursor()
    cursor.execute("SELECT content FROM memory ORDER BY created_at DESC LIMIT ?", (limit,))
    rows = cursor.fetchall()
    conn.close()
    return [r[0] for r in rows]

def fetch_news():
    feed = feedparser.parse("http://feeds.bbci.co.uk/news/rss.xml")
    return [{"title": entry.title, "link": entry.link} for entry in feed.entries[:5]]

def extract_text(url):
    try:
        res = requests.get(url, timeout=5)
        soup = BeautifulSoup(res.text, "html.parser")
        paragraphs = soup.find_all("p")
        return " ".join([p.get_text() for p in paragraphs])[:3000]
    except: return ""

class NewsAgent():
    def __init__(self):
        self.client = genai.Client(api_key=os.environ.get("GEMINI_API_KEY"))
        self.model = "gemini-2.5-flash"

    def think(self, article_text, title):
        memory = "\n".join(load_memory())
        prompt = f"Existing knowledge:\n{memory}\n\nRead this: {title}\n{article_text}\nReturn 'SKIP' or insight."
        contents = [types.Content(role="user", parts=[types.Part.from_text(text=prompt)])]
        config = types.GenerateContentConfig(thinking_config=types.ThinkingConfig(thinking_budget=0))
        response = ""
        for chunk in self.client.models.generate_content_stream(model=self.model, contents=contents, config=config):
            if chunk.text: response += chunk.text
        return response.strip()

def run_pipeline():
    init_db()
    agent = NewsAgent()
    articles = fetch_news()
    for art in articles:
        text = extract_text(art["link"])
        if not text: continue
        insight = agent.think(text, art["title"])
        if insight != "SKIP": save_to_db(insight, art["link"])

if __name__ == "__main__": run_pipeline()
Mental Model

Lines 9-33: We set up the SQLite engine. Unlike standard memory, this database persists on disk, allowing the agent to remember things across separate runs.

Lines 35-46: The "Scraper" logic. We use feedparser to grab headers and BeautifulSoup to pull the raw body text from the web.

Lines 52-65: The Agent's brain. Crucially, it calls load_memory() before thinking. This provides context, so the agent can avoid saving information it already knows.

Lines 67-77: The Pipeline. It acts as a filter: Scraping → Analyzing → Selective Saving. This is the difference between a "scraper" and an "agent."
