Pattern 9a – Long‑Term Memory Agent

This pattern illustrates how an agent can develop long‑term memory by reading external news articles, deciding whether an insight is important, and saving valuable information into a SQLite database. Over time, the agent builds a knowledge base to enhance future analysis. 

1# install dependencies (e.g. requests, beautifulsoup4, feedparser)
2import os
3import sqlite3
4import requests
5import feedparser
6from bs4 import BeautifulSoup
7from google import genai
8from google.genai import types

11# initialize the database
12def init_db():
13    conn = sqlite3.connect("agent_memory.db")
14    cursor = conn.cursor()
15    cursor.execute("""
16    CREATE TABLE IF NOT EXISTS memory (
17        id INTEGER PRIMARY KEY AUTOINCREMENT,
18        content TEXT,
19        source TEXT,
20        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
21    )
22    """)
23    conn.commit()
24    conn.close()

27def save_to_db(content, source):
28    conn = sqlite3.connect("agent_memory.db")
29    cursor = conn.cursor()
30    cursor.execute("""
31        INSERT INTO memory (content, source)
32        VALUES (?, ?)
33    """, (content, source))
34    conn.commit()
35    conn.close()

38def load_memory(limit=5):
39    conn = sqlite3.connect("agent_memory.db")
40    cursor = conn.cursor()
41    cursor.execute("""
42        SELECT content FROM memory
43        ORDER BY created_at DESC
44        LIMIT ?
45    """, (limit,))
46    rows = cursor.fetchall()
47    conn.close()
48    return [r[0] for r in rows]

52# fetch top news items via RSS
53def fetch_news():
54    feed = feedparser.parse("http://feeds.bbci.co.uk/news/rss.xml")
55    articles = []
56    for entry in feed.entries[:5]:
57        articles.append({
58            "title": entry.title,
59            "link": entry.link
60        })
61    return articles

65# extract article text from web
66def extract_text(url):
67    try:
68        res = requests.get(url, timeout=5)
69        soup = BeautifulSoup(res.text, "html.parser")
70        paragraphs = soup.find_all("p")
71        text = " ".join([p.get_text() for p in paragraphs])
72        return text[:3000]  # trim to avoid token overflow
73    except:
74        return ""

100# Agent that extracts insights from articles
101class NewsAgent:
102    def __init__(self):
103        self.client = genai.Client(api_key=os.environ.get("GEMINI_API_KEY"))
104        self.model = "gemini-2.5-flash"
105    def think(self, article_text, title):
107        memory = "\n".join(load_memory())
108        prompt = f"""
109You are an intelligent analyst.
110
111Existing knowledge:
112{memory}
113
114Read this news and extract ONLY important insights worth remembering.
115
116Rules:
117- Ignore noise
118- Keep it short (1-2 lines)
119- Only store meaningful facts or trends
120
121News Title:
122{title}
123
124News Content:
125{article_text}
126
127Return:
128- "SKIP" if nothing important
129- Otherwise return insight
130"""
138        contents = [
139            types.Content(role="user", parts=[types.Part.from_text(text=prompt)])
140        ]
144        config = types.GenerateContentConfig(
145            thinking_config=types.ThinkingConfig(thinking_budget=0)
146        )
148        response = ""
150        for chunk in self.client.models.generate_content_stream(
151            model=self.model, contents=contents, config=config
152        ):
153            if chunk.text:
154                print(chunk.text, end="")
155                response += chunk.text
156        print("\n")
158        return response.strip()

165# Pipeline: fetch, analyse, save
166def run_pipeline():
167    init_db()
168    agent = NewsAgent()
170    articles = fetch_news()
172    for art in articles:
173        print(f"\n📰 Reading: {art['title']}")
174        text = extract_text(art["link"])
176        if not text:
177            print("Failed to extract text")
178            continue
182        insight = agent.think(text, art["title"])
184        if insight != "SKIP":
185            save_to_db(insight, art["link"])
186            print("Saved to DB")
187        else:
188            print("Skipped")

194run_pipeline()

Building Long‑Term Memory

This script constructs an agent that can read external news feeds, distil them into concise insights, and persist those insights in a database. It demonstrates how long‑term memory can enhance a retrieval‑augmented agent by providing context from previous observations.

Imports and Setup

The code begins by importing modules for HTTP requests, RSS parsing, HTML extraction (BeautifulSoup), database access (sqlite3), and the Gemini API. These libraries support fetching news, scraping article text, and interacting with the model.

Database Functions

  • init_db() (lines 12‑24) opens a SQLite database file (agent_memory.db) and ensures a table named memory exists. Each record stores the insight text, its source URL, and a timestamp.
  • save_to_db() (lines 27‑35) inserts a new insight into the database. The function accepts the text and source and commits the transaction.
  • load_memory() (lines 38‑48) retrieves the most recent n insights (default five) from the database. The returned list is later used to provide context when summarising new articles.

Fetching and Parsing News

  • fetch_news() (lines 53‑61) uses feedparser to read the BBC News RSS feed. It gathers the titles and links for the first five entries and returns them as a list of dictionaries.
  • extract_text() (lines 66‑74) downloads the article HTML and extracts all paragraph text into a single string. It trims the output to 3000 characters to avoid exceeding token limits. If the request fails, it returns an empty string.

The NewsAgent

The NewsAgent class (lines 101‑158) holds a Gemini client and defines a think() method to summarise articles:

  • It concatenates recent memory entries using \n.join(load_memory()) (line 107). This ensures the agent remembers past insights when evaluating new articles.
  • A prompt is assembled (lines 108‑130) that instructs the model to ignore noise and return only short, meaningful insights. If nothing important is found, the model should respond with the literal string "SKIP".
  • The prompt is sent via generate_content_stream() (lines 150‑155), streaming the response to stdout and concatenating it into a final string. The method returns the trimmed response.

Pipeline Execution

The run_pipeline() function (lines 166‑187) ties everything together. It initialises the database, instantiates a NewsAgent, and fetches the latest news articles. For each article it:

  • Prints the title to provide context.
  • Extracts the full article text; if extraction fails, it skips to the next article.
  • Asks the agent for an insight; if the response is not "SKIP", it saves the insight to the database and indicates success, otherwise it notes that nothing worth storing was found.

Finally, line 194 calls run_pipeline() so the process runs as soon as the script is executed. Over time the database accumulates insights that can be retrieved and used by future agents or queries.
