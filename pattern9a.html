OVERVIEW

What this pattern teaches

The Long-Term Memory Agent (Pattern 10) demonstrates how to bridge external data sources (RSS feeds) with persistent storage (SQLite). The agent evaluates news articles not just in isolation, but by checking its existing knowledge base first.

This pattern moves beyond simple RAG (Retrieval Augmented Generation) by giving the agent Agency over its own memory—deciding what is "meaningful" to save and what to ignore, building a custom intelligence layer.

CODE

Line-by-line walkthrough

pattern10.py 
import sqlite3
import feedparser
from google import genai

# DB Functions
def init_db():
    conn = sqlite3.connect("agent_memory.db")
    conn.execute("CREATE TABLE IF NOT EXISTS memory (content TEXT)")
    conn.close()

def save_to_db(content):
    conn = sqlite3.connect("agent_memory.db")
    conn.execute("INSERT INTO memory VALUES (?)", (content,))
    conn.commit()
    conn.close()

class NewsAgent:
    def __init__(self):
        self.client = genai.Client()
        self.model = "gemini-2.5-flash"

    def think(self, article, title):
        # Load previous context
        memory = self.get_recent_memory()
        prompt = f"Knowledge: {memory}\n\nAnalyze: {title}\n{article}\nReturn 'SKIP' or insight."
        
        res = self.client.models.generate_content(model=self.model, contents=prompt)
        return res.text.strip()

    def get_recent_memory(self):
        conn = sqlite3.connect("agent_memory.db")
        rows = conn.execute("SELECT content FROM memory LIMIT 5").fetchall()
        return "\n".join([r[0] for r in rows])

def run():
    init_db()
    agent = NewsAgent()
    feed = feedparser.parse("http://feeds.bbci.co.uk/news/rss.xml")
    
    for entry in feed.entries[:3]:
        insight = agent.think(entry.summary, entry.title)
        if insight != "SKIP":
            save_to_db(insight)
Mental Model

Lines 6-15: We use sqlite3 to create a local persistent file. Unlike Python's memory, this data survives even after the script stops running.

Lines 24-29: Before analyzing new news, the agent calls get_recent_memory(). This injects "what it already knows" into the prompt, preventing duplicate work.

Lines 41-48: The pipeline loop. The agent acts as a gatekeeper—if it returns "SKIP", the database remains untouched. Only "meaningful" insights are committed.

Long-term Value: Over days of running this, the agent builds a specialized context window that is unique to the data it has encountered.
