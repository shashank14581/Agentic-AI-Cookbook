Pattern 9e – Vector Search Before Generation

Instead of chunking documents, this pattern embeds entire sentences or facts and retrieves the most similar ones before answering. The agent then answers strictly from the retrieved documents.

10class SimpleEmbeddingModel(nn.Module):
11    def __init__(self, vocab_size, embedding_dim=32):
12        super().__init__()
13        self.embedding = nn.Embedding(vocab_size, embedding_dim)
15    def forward(self, token_ids):
16        word_embeddings = self.embedding(token_ids)
17        return word_embeddings.mean(dim=0)

20class VectorSearchIndex:
21    def __init__(self):
22        self.documents = []
24    def add(self, doc_id, text, embedding):
25        self.documents.append({"doc_id": doc_id, "text": text, "embedding": embedding})
31    def search(self, query_embedding, top_k=3):
32        results = []
34        for doc in self.documents:
35            similarity = F.cosine_similarity(query_embedding.unsqueeze(0), doc["embedding"].unsqueeze(0)).item()
40            results.append((similarity, doc))
42        results = sorted(results, key=lambda x: x[0], reverse=True)
43        return results[:top_k]

46class VectorSearchMemory:
47    def __init__(self, embedding_dim=32):
48        self.word_to_id = {"": 0}
49        self.embedding_dim = embedding_dim
50        self.model = None
51        self.index = VectorSearchIndex()
53    def build_vocab(self, texts):
54        for text in texts:
55            for word in text.lower().split():
56                if word not in self.word_to_id:
57                    self.word_to_id[word] = len(self.word_to_id)
59    def encode_text(self, text):
60        token_ids = [self.word_to_id.get(word, 0) for word in text.lower().split()]
65        return torch.tensor(token_ids, dtype=torch.long)
67    def build_index(self, texts):
68        self.build_vocab(texts)
70        self.model = SimpleEmbeddingModel(vocab_size=len(self.word_to_id), embedding_dim=self.embedding_dim)
75        for i, text in enumerate(texts, start=1):
76            token_ids = self.encode_text(text)
78            with torch.no_grad():
79                embedding = self.model(token_ids)
81            self.index.add(doc_id=f"doc_{i}", text=text, embedding=embedding)
87            print(f"Indexed doc_{i}")
89    def retrieve(self, query, top_k=3):
90        token_ids = self.encode_text(query)
92        with torch.no_grad():
93            query_embedding = self.model(token_ids)
95        results = self.index.search(query_embedding, top_k=top_k)
97        context = ""
99        for score, doc in results:
100            context += f"""
101Document: {doc["doc_id"]}
102Similarity: {round(score, 4)}
103Text: {doc["text"]}
104"""
106        return context

Vector Search Before Generation

This example demonstrates a simpler RAG pipeline that embeds entire sentences or facts rather than splitting them into multiple chunks. It illustrates how an agent can retrieve documents by similarity and then answer questions strictly from those documents.

Embedding and Indexing

  • SimpleEmbeddingModel (lines 10‑17) maps words to vectors and returns the mean embedding for a sequence of tokens.
  • VectorSearchIndex (lines 20‑43) stores document embeddings and performs a cosine similarity search to rank documents by their relevance to a query embedding.
  • VectorSearchMemory (lines 46‑106) manages vocabulary building (build_vocab()), encodes text into token IDs, builds the embedding model and index, and retrieves the top k documents for a query. The retrieve() method returns a formatted context string showing document IDs, similarity scores and text.

Agent Logic

The BaseAgent class (lines 109‑167) uses VectorSearchMemory to get context for a user question. It builds a prompt that clearly outlines a two‑step process: first retrieve relevant documents, then answer using only those documents. The agent instructs Gemini not to guess and to admit when there isn’t enough context.

VectorSearchAgent (lines 170‑182) simply provides a system prompt reminding the model of its constraints.

Example Execution

In the main block (lines 192‑224), a list of private facts about Project Sarayu and Blue Mango is indexed. A VectorSearchAgent is then asked several questions about the project, and for each question, the agent retrieves three relevant facts and answers using only those facts. This pattern emphasises faithful use of retrieved knowledge and prevents hallucination.
