What this pattern teaches

The Vector Search Agent (Pattern 14) evolves the RAG architecture by focusing on Semantic Similarity. Instead of simple text matching, we represent the "Project Sarayu" knowledge base as high-dimensional vectors.

The implementation uses SimpleEmbeddingModel to create vector identities for private company secrets. When a user asks about "Blue Mango," the agent retrieves the most mathematically similar documents to provide a strictly grounded response.

import os, torch
import torch.nn as nn
import torch.nn.functional as F
from google import genai
from google.genai import types

class SimpleEmbeddingModel(nn.Module):
    def __init__(self, vocab_size, embedding_dim=32):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embedding_dim)
    def forward(self, token_ids):
        return self.embedding(token_ids).mean(dim=0)

class VectorSearchIndex:
    def __init__(self): self.documents = []
    def add(self, doc_id, text, embedding):
        self.documents.append({"doc_id": doc_id, "text": text, "embedding": embedding})
    def search(self, query_emb, top_k=3):
        results = []
        for doc in self.documents:
            sim = F.cosine_similarity(query_emb.unsqueeze(0), doc["embedding"].unsqueeze(0)).item()
            results.append((sim, doc))
        return sorted(results, key=lambda x: x[0], reverse=True)[:top_k]

class VectorSearchMemory:
    def __init__(self, embedding_dim=32):
        self.word_to_id = {"<UNK>": 0}
        self.index = VectorSearchIndex()
    def build_index(self, texts):
        for text in texts:
            for w in text.lower().split():
                if w not in self.word_to_id: self.word_to_id[w] = len(self.word_to_id)
        self.model = SimpleEmbeddingModel(len(self.word_to_id), 32)
        for i, text in enumerate(texts, start=1):
            ids = torch.tensor([self.word_to_id.get(w, 0) for w in text.lower().split()], dtype=torch.long)
            self.index.add(f"doc_{i}", text, self.model(ids))

Mental Model

Lines 7-13: The SimpleEmbeddingModel represents words as vectors. By calculating the mean of word vectors, we get a single mathematical "fingerprint" for a whole sentence.

Lines 15-25: The VectorSearchIndex performs a linear scan of all documents, calculating Cosine Similarity. It returns the documents with the highest semantic overlap with the query.

Lines 27-41: VectorSearchMemory automates the pipeline: building a vocabulary, initializing the neural weights, and converting the "Project Sarayu" facts into a searchable vector space.

Lines 43-48: The VectorSearchAgent acts as a gatekeeper. It retrieves the "Top 3" most relevant facts and passes them to Gemini with a strict instruction: "If the answer isn't here, do not guess."
