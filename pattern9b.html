Pattern 9b – Shared Memory Agents

In this pattern, multiple agents share a single memory store. Each agent can read everything written to the store and append their own contributions. The conversation evolves as agents incorporate shared context into their responses. 

1import os
2from google import genai
3from google.genai import types
4os.environ["GEMINI_API_KEY"] = "your_api_key"

7class SharedMemory:
8    def __init__(self):
9        self.store = []
11    def write(self, agent_name, input_text, output_text):
12        self.store.append({
13            "agent": agent_name,
14            "input": input_text,
15            "output": output_text
16        })
18    def read(self):
19        if not self.store:
20            return "No shared memory yet."
22        memory_text = ""
23        for i, item in enumerate(self.store, start=1):
24            memory_text += f"""
25Memory {i}
26Agent: {item['agent']}
27Input: {item['input']}
28Output: {item['output']}
29"""
30        return memory_text

33class BaseAgent:
34    def __init__(self, name, sys_prompt, shared_memory, model="gemini-2.5-flash-lite"):
35        self.name = name
36        self.sys_prompt = sys_prompt
37        self.shared_memory = shared_memory
38        self.client = genai.Client(api_key=os.environ.get("GEMINI_API_KEY"))
39        self.model = model

41    def think(self, input_text):
43        memory_context = self.shared_memory.read()
45        final_input = f"""
46You have access to shared memory.
47
48Shared Memory:
49{memory_context}
51Current Input:
52{input_text}
54Respond as {self.name}.
55"""
57        contents = [types.Content(role="user", parts=[types.Part.from_text(text=final_input)])]
64        config = types.GenerateContentConfig(
65            thinking_config=types.ThinkingConfig(thinking_budget=0),
66            system_instruction=[types.Part.from_text(text=self.sys_prompt)]
67        )
69        response = ""
71        for chunk in self.client.models.generate_content_stream(model=self.model, contents=contents, config=config):
72            if chunk.text:
73                print(chunk.text, end="")
74                response += chunk.text
80        print()
82        self.shared_memory.write(agent_name=self.name, input_text=input_text, output_text=response)
88        return response

91class Human(BaseAgent):
92    def __init__(self, name, shared_memory):
93        super().__init__(name, f"You are {name}, a human. Talk naturally.", shared_memory)

100class Dog(BaseAgent):
101    def __init__(self, name, shared_memory):
103        super().__init__(name, f"You are {name}, a dog. Bark, be playful.", shared_memory)

109def orchestrate(human, dog, turns=5):
110    message = "Hi Dog!"
112    for _ in range(turns):
113        print(f"\n{human.name}: ", end="")
114        human_out = human.think(message)
116        print(f"\n{dog.name}: ", end="")
117        dog_out = dog.think(human_out)
119        message = dog_out

123    shared_memory = SharedMemory()
125    human = Human("Shashank", shared_memory)
126    dog = Dog("Bruno", shared_memory)
128    orchestrate(human, dog)

Sharing Memory Across Agents

Shared memory allows agents to build on each other’s experiences. Rather than storing knowledge privately, all interactions are saved into a common store, which becomes part of every new prompt. This pattern encourages emergent behaviour as agents incorporate the entire conversation history into their reasoning.

The SharedMemory store

Lines 7‑30 define a lightweight in‑memory database. The store list holds dictionaries for each interaction. write() (lines 11‑16) appends a record with the agent’s name, the input it received, and the output it produced. read() (lines 18‑30) concatenates all records into a formatted string. If nothing has been written yet, it returns a friendly notice.

BaseAgent with Memory Access

The BaseAgent (lines 33‑88) takes a shared_memory instance and a system prompt. In think(), it reads the shared memory into memory_context (line 43) and builds a final prompt (lines 45‑55) containing:

  • A notice that the agent can access shared memory.
  • The entire shared memory contents, providing context.
  • The current input from the other participant.
  • A final instruction to respond in character as self.name.

The prompt is sent to Gemini using generate_content_stream() (lines 71‑74). Once a complete response is assembled, it is printed and added back into the shared memory via write() (line 82).

Specialised Humans and Dogs

Two subclasses provide system prompts tailored to a human and a dog. The Human (lines 91‑93) speaks naturally, while the Dog (lines 100‑103) barks and plays. Both pass the same shared memory reference to their base class.

Orchestration

The orchestrate() function (lines 109‑119) demonstrates how memory accumulates. It starts with a simple greeting from the human to the dog. For each of the specified turns, the human responds to the latest message and writes the conversation to shared memory. The dog reads this updated memory when formulating its reply, and so on. By the last turn, both agents have access to the full conversation history.

Execution

When run as a script, the code creates a single SharedMemory instance, instantiates a human named “Shashank” and a dog named “Bruno”, and runs the orchestrated conversation. Because both agents read from and write to the same memory store, their responses reflect the evolving context.
