Pattern 6: Reasoning and Handoff

Agents reason about their response and decide who speaks next, returning structured JSON.

Summary

This pattern introduces the concept of a handoff agent. Each agent still derives from a common base class, but instead of returning plain text, it returns a JSON object containing its message, the next agent to speak (handoff_to), and a brief reason. The orchestrator no longer decides who goes next—it simply reads the handoff_to field and routes the conversation accordingly. If JSON parsing fails, a fallback ensures the conversation continues.


import os
import json
from google import genai
from google.genai import types

# Base agent remains similar: stores name, prompt, memory and Gemini client.
class BaseAgent():
    def __init__(self, name, sys_prompt, model="gemini-2.5-flash"):
        self.name = name
        self.sys_prompt = sys_prompt
        self.memory = []
        self.client = genai.Client(api_key=os.environ.get("GEMINI_API_KEY"))
        self.model = model
    def think(self, input_text):
        if not input_text:
            input_text = "continue"
        # Build Gemini request and stream response
        contents = [
            types.Content(role="user", parts=[types.Part.from_text(text=input_text)])
        ]
        config = types.GenerateContentConfig(
            thinking_config=types.ThinkingConfig(thinking_budget=0),
            system_instruction=[types.Part.from_text(text=self.sys_prompt)]
        )
        response = ""
        for chunk in self.client.models.generate_content_stream(
            model=self.model, contents=contents, config=config
        ):
            if chunk.text:
                response += chunk.text
        self.memory.append((input_text, response))
        return response

# HandoffAgent: returns JSON with message, handoff_to, and reason.
class HandoffAgent(BaseAgent):
    def __init__(self, name, persona, peers):
        peer_list = ", ".join(peers)
        super().__init__(
            name,
            f"""{persona}

You are part of a multi-agent conversation. After responding, you must decide who speaks next.

Available agents to hand off to: {peer_list}

You MUST return ONLY a JSON object with fields:
- message: your reply
- handoff_to: one of {peer_list}
- reason: one sentence justification
"""
        )
    def think_and_handoff(self, input_text):
        raw = self.think(input_text)
        # Remove markdown fences if present
        clean = raw.strip()
        if clean.startswith("```"):
            parts = clean.split("```")
            clean = parts[1] if len(parts) > 1 else clean
            if clean.startswith("json"):
                clean = clean[4:]
        clean = clean.strip()
        try:
            parsed = json.loads(clean)
            return parsed
        except json.JSONDecodeError:
            # Fallback when parsing fails
            return {
                "message": raw,
                "handoff_to": None,
                "reason": "parse error — orchestrator will decide next agent"
            }

# Specialized agents with different personas
class Human(HandoffAgent):
    def __init__(self, name, peers):
        super().__init__(name, f"You are {name}, a human. Talk naturally and curiously.", peers)

class Dog(HandoffAgent):
    def __init__(self, name, peers):
        super().__init__(name, f"You are {name}, a playful dog. You bark, wag, and are enthusiastic.", peers)

class Vet(HandoffAgent):
    def __init__(self, name, peers):
        super().__init__(name, f"You are {name}, a veterinarian. You give calm, expert advice about animals.", peers)

# Orchestrator: reads handoff_to and routes accordingly
def orchestrate(agents, starter_name, opening_message, turns=6):
    current_name = starter_name
    message = opening_message
    for i in range(turns):
        current_agent = agents[current_name]
        result = current_agent.think_and_handoff(message)
        print(f"{current_name}: {result.get('message', '')}")
        print(f"→ Handing off to: {result.get('handoff_to')} | Reason: {result.get('reason')}")
        next_name = result.get("handoff_to")
        if not next_name or next_name not in agents:
            print(f"Invalid handoff target '{next_name}'. Stopping.")
            break
        message = result.get("message", "continue")
        current_name = next_name

# Main: set up agents and start conversation
if __name__ == "__main__":
    human = Human("Shashank", peers=["Bruno", "DrMehra"])
    dog   = Dog("Bruno",      peers=["Shashank", "DrMehra"])
    vet   = Vet("DrMehra",    peers=["Shashank", "Bruno"])
    agents = {
        "Shashank": human,
        "Bruno":    dog,
        "DrMehra":  vet
    }
    orchestrate(agents, starter_name="Shashank", opening_message="Bruno hasn't been eating well lately.", turns=6)

BaseAgent: The base class encapsulates common behavior: storing the agent’s name, system prompt, memory, and a Gemini client. The think() method constructs a request, streams the model’s response, appends the conversation to memory, and returns the text.

HandoffAgent: Inherits from BaseAgent and enforces a structured JSON output. Its system prompt lists available peers and instructs the model to return a JSON object with message, handoff_to, and reason. The think_and_handoff() method parses the JSON and falls back gracefully if parsing fails.

Specialized Agents: Three subclasses—Human, Dog, and Vet—define different personas via their prompts and pass along the list of peers who can speak next.

Orchestrator: Instead of deciding the next speaker, the orchestrator reads the handoff_to field from the agent’s JSON response. It prints the agent’s message and handoff decision, then routes the conversation. If the agent hands off to an invalid peer, the conversation stops.

Main: Sets up the agents (Shashank, Bruno, Dr Mehra) with appropriate peers and kicks off the conversation with a concern about Bruno’s eating habits. Each agent thinks, speaks, and decides who should speak next, creating a dynamic, self‑routing dialog.
