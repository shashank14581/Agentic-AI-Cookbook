Pattern 6 · Handoff

Reasoning and Handoff

In this pattern, agents do not just respond. They reason about who should speak next and return structured JSON containing a message, a handoff target, and a reason.

The agent passes the baton.

Control flow moves from the orchestrator into the agent response itself.

Agent speaks
Agent chooses next speaker
Orchestrator follows the handoff
Summary Code Walkthrough Next Pattern
OVERVIEW

What this pattern teaches

This pattern introduces the handoff agent. Each agent still derives from a common base class, but instead of returning plain text, it returns a JSON object containing message, handoff_to, and reason.

The orchestrator no longer decides who goes next. It simply reads the handoff_to field and routes the conversation accordingly. If JSON parsing fails, a fallback ensures the conversation can stop safely.

import os
import json
from google import genai
from google.genai import types

# Base agent remains similar.
class BaseAgent():

    def __init__(self, name, sys_prompt, model="gemini-2.5-flash"):

        self.name = name
        self.sys_prompt = sys_prompt
        self.memory = []

        self.client = genai.Client(
            api_key=os.environ.get("GEMINI_API_KEY")
        )

        self.model = model

    def think(self, input_text):

        if not input_text:
            input_text = "continue"

        contents = [
            types.Content(
                role="user",
                parts=[types.Part.from_text(text=input_text)]
            )
        ]

        config = types.GenerateContentConfig(
            thinking_config=types.ThinkingConfig(thinking_budget=0),
            system_instruction=[types.Part.from_text(text=self.sys_prompt)]
        )

        response = ""

        for chunk in self.client.models.generate_content_stream(
            model=self.model,
            contents=contents,
            config=config
        ):

            if chunk.text:
                response += chunk.text

        self.memory.append((input_text, response))

        return response

# HandoffAgent returns JSON with message, handoff_to, and reason.
class HandoffAgent(BaseAgent):

    def __init__(self, name, persona, peers):

        peer_list = ", ".join(peers)

        super().__init__(
            name,
            f"""{persona}

You are part of a multi-agent conversation.
After responding, you must decide who speaks next.

Available agents to hand off to: {peer_list}

You MUST return ONLY a JSON object with fields:
- message: your reply
- handoff_to: one of {peer_list}
- reason: one sentence justification
"""
        )

    def think_and_handoff(self, input_text):

        raw = self.think(input_text)

        clean = raw.strip()

        if clean.startswith("```"):

            parts = clean.split("```")
            clean = parts[1] if len(parts) > 1 else clean

            if clean.startswith("json"):
                clean = clean[4:]

        clean = clean.strip()

        try:

            parsed = json.loads(clean)

            return parsed

        except json.JSONDecodeError:

            return {
                "message": raw,
                "handoff_to": None,
                "reason": "parse error — orchestrator will decide next agent"
            }

# Specialized agents with different personas.
class Human(HandoffAgent):

    def __init__(self, name, peers):

        super().__init__(
            name,
            f"You are {name}, a human. Talk naturally and curiously.",
            peers
        )

class Dog(HandoffAgent):

    def __init__(self, name, peers):

        super().__init__(
            name,
            f"You are {name}, a playful dog. You bark, wag, and are enthusiastic.",
            peers
        )

class Vet(HandoffAgent):

    def __init__(self, name, peers):

        super().__init__(
            name,
            f"You are {name}, a veterinarian. You give calm, expert advice about animals.",
            peers
        )

# Orchestrator reads handoff_to and routes accordingly.
def orchestrate(agents, starter_name, opening_message, turns=6):

    current_name = starter_name
    message = opening_message

    for i in range(turns):

        current_agent = agents[current_name]

        result = current_agent.think_and_handoff(message)

        print(f"{current_name}: {result.get('message', '')}")

        print(
            f"→ Handing off to: {result.get('handoff_to')} | "
            f"Reason: {result.get('reason')}"
        )

        next_name = result.get("handoff_to")

        if not next_name or next_name not in agents:

            print(f"Invalid handoff target '{next_name}'. Stopping.")

            break

        message = result.get("message", "continue")

        current_name = next_name

# Main
if __name__ == "__main__":

    human = Human("Shashank", peers=["Bruno", "DrMehra"])
    dog = Dog("Bruno", peers=["Shashank", "DrMehra"])
    vet = Vet("DrMehra", peers=["Shashank", "Bruno"])

    agents = {
        "Shashank": human,
        "Bruno": dog,
        "DrMehra": vet
    }

    orchestrate(
        agents,
        starter_name="Shashank",
        opening_message="Bruno hasn't been eating well lately.",
        turns=6
    )
Mental Model

Lines 1–4: Import os, json, and the Gemini SDK. The JSON module is needed because each agent now returns structured routing data.

Lines 7–47: Define the shared BaseAgent. It stores the agent name, prompt, memory, Gemini client, and exposes a reusable think() method.

Lines 50–75: Define HandoffAgent. It builds a prompt that forces the model to return a JSON object with a reply, a next speaker, and a reason.

Lines 77–105: think_and_handoff() calls the model, cleans possible markdown fences, parses JSON, and returns a fallback object if parsing fails.

Lines 108–140: Define specialized handoff agents: Human, Dog, and Vet. Each has its own persona and peer list.

Lines 143–176: The orchestrator reads handoff_to from the agent’s JSON response and routes control to that next agent.

Lines 179–196: Create the three agents, register them in a dictionary, and start the conversation with a concern about Bruno not eating well.
