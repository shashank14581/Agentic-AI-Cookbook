OVERVIEW

What this pattern teaches

Pattern 20 is the server-side counterpart to Pattern 19. Pattern 19 showed how a client calls an existing MCP endpoint. This pattern shows how that endpoint is created in the first place using FastMCP.

The core sequence is: install MCP → define server → register tool → choose port → start streamable HTTP server → reuse if already running. This is the minimum reusable tool-server pattern for agentic systems.

import time
import socket
import threading
import nest_asyncio

from mcp.server.fastmcp import FastMCP

nest_asyncio.apply()


PORT = 8012

mcp = FastMCP(
    name="simple_tool_server",
    stateless_http=True,
    json_response=True
)


@mcp.tool()
def search_private_knowledge(query: str) -> str:
    query = query.lower()

    if "blue mango" in query:
        return "Blue Mango is the internal codename for the customer retention model in Project Sarayu."

    if "silver gate" in query:
        return "Silver Gate is the manual review queue for Project Sarayu."

    return "No matching private knowledge found."


def run_mcp_server():
    mcp.settings.port = PORT
    mcp.run(transport="streamable-http")


def port_is_busy(port):
    with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as s:
        return s.connect_ex(("127.0.0.1", port)) == 0


if not port_is_busy(PORT):
    server_thread = threading.Thread(
        target=run_mcp_server,
        daemon=True
    )
    server_thread.start()
    time.sleep(3)
    print(f"MCP server started on port {PORT}.")
else:
    print(f"MCP server already running on port {PORT}.")
Mental Model

Line 1: Installs MCP, Uvicorn, and nest_asyncio. This keeps the script notebook-friendly.

Lines 3-10: Imports utilities for sleeping, checking ports, running the server in a background thread, and patching the notebook event loop.

Line 13: Chooses port 8012. Clients will connect to this server through http://127.0.0.1:8012/mcp.

Lines 15-19: Creates a FastMCP server named simple_tool_server. Stateless HTTP and JSON response mode make it easier to call from clients.

Lines 22-32: Registers search_private_knowledge as an MCP tool. The decorator makes the function available to MCP clients.

Lines 35-37: Defines how to run the server. The transport is streamable-http, which matches the client pattern from Pattern 19.

Lines 40-42: Checks whether the port is already occupied. This prevents duplicate-server crashes when rerunning cells.

Lines 45-55: Starts the MCP server in a daemon thread if the port is free. If the server already exists, the script reuses it.
