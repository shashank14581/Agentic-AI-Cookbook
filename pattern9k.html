Pattern 9k – Creating Your Own MCP Tool Server

Build a custom MCP server that exposes tools for agents to call. This pattern focuses on the server side and does not involve Gemini.

Server Setup

The script begins by importing the necessary modules, applying nest_asyncio and specifying a port (8012). It then creates a FastMCP instance with a human‑readable name. The server is configured for stateless HTTP transport and JSON responses.

Defining Tools

Using the @mcp.tool() decorator, you can expose functions over HTTP. Here, search_private_knowledge checks for keywords in the query and returns a canned answer if it finds “Blue Mango” or “Silver Gate”. Otherwise it returns a default message.

Starting the Server

The helper function port_is_busy() prevents multiple servers from binding to the same port. If the port is available, the server is started on a background thread and left running. You can add more tools by decorating additional functions before launching the server.
