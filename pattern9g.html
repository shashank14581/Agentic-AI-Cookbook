Pattern 9g – Agentic RAG

Here the agent decides on its own whether to perform retrieval. It combines a router for decision making with a retrieval‑augmented memory.

26     def build_vocab(self, texts):

28         for word in text.lower().split():

38     def build_index(self, knowledge_base):
39       self.build_vocab(knowledge_base)
40       self.model = SimpleEmbeddingModel(
41         len(self.word_to_id),
42         self.embedding_dim
43       )
44       for i, text in enumerate(knowledge_base, start=1):
45         token_ids = self.encode_text(text)
46         with torch.no_grad():
47           embedding = self.model(token_ids)
48         self.store.append({"doc_id":f"doc_{i}","text":text,"embedding":embedding})
49         print(f"Indexed doc_{i}")

51     def retrieve(self, query, top_k=3):
52       query_ids = self.encode_text(query)
53       with torch.no_grad():
54         query_embedding = self.model(query_ids)
55       results = []
56       for item in self.store:
57         score = F.cosine_similarity(
58             query_embedding.unsqueeze(0),
59             item["embedding"].unsqueeze(0)
60         ).item()
61         results.append((score, item))
62       results = sorted(results, key=lambda x: x[0], reverse=True)
63       context = ""
64       for score, item in results[:top_k]:
65         context += f"""
66   Document: {item["doc_id"]}
67   Similarity: {round(score, 4)}
68   Text: {item["text"]}
69   """
70       return context

Decision Agent

The RetrievalDecisionAgent is a lightweight agent whose job is to answer a single question: should we retrieve private knowledge? Its system prompt instructs Gemini to return either RETRIEVE or NO_RETRIEVE, based on whether the question refers to private projects, codenames or operational details. The method should_retrieve() normalises the model’s response and returns a boolean flag.

RAG Memory

The RAG memory is the same as in the basic pattern: it builds a vocabulary, computes document embeddings with a tiny neural network, and retrieves the most similar entries using cosine similarity.

Agentic Flow

The AgenticRAGAgent first consults the decision agent. If retrieval is required, it prints [Decision: RETRIEVE], fetches context from the RAG memory and constructs a prompt that forces the model to answer only from that context. Otherwise it prints [Decision: NO_RETRIEVE] and asks Gemini to reason normally. In either case the result is generated via the base agent’s think() method.

Example Execution

The provided questions demonstrate both branches. General questions like “What is RAG?” do not require retrieval, while internal queries about Blue Mango or its confidence thresholds trigger retrieval. This pattern illustrates how to insert conditional logic into a RAG workflow.
