Pattern 9 – Debate Pattern

This pattern pits two agents against each other in an adversarial debate. Each debater takes a position on a topic, attacks their opponent, and a judge delivers a final verdict. The implementation uses Python’s asyncio to run conversations concurrently and demonstrates how structured multi‑stage interactions can produce richer reasoning. 

1import os
2import asyncio
3from google import genai
4from google.genai import types

6# ── Base Agent ────────────────────────────────────────────────────────────────
7class BaseAgent:
8    def __init__(self, name: str, sys_prompt: str, model: str = "gemini-2.5-flash-lite"):
9        self.name = name
10        self.sys_prompt = sys_prompt
11        self.memory: list[tuple[str, str]] = []
12        self.client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])
13        self.model = model
14        self._config = types.GenerateContentConfig(
15            thinking_config=types.ThinkingConfig(thinking_budget=0),
16            system_instruction=[types.Part.from_text(text=self.sys_prompt)],
17        )

18    async def think(self, input_text: str) -> str:
19        contents = [
20            types.Content(
21                role="user",
22                parts=[types.Part.from_text(text=input_text)],
23            )
24        ]
25        response = ""
26        stream = await self.client.aio.models.generate_content_stream(
27            model=self.model,
28            contents=contents,
29            config=self._config,
30        )
31        async for chunk in stream:
32            if chunk.text:
33                print(chunk.text, end="", flush=True)
34                response += chunk.text
35        print()
36        self.memory.append((input_text, response))
37        return response

41# ── Agent Roles ───────────────────────────────────────────────────────────────
42class Debater(BaseAgent):
43    def __init__(self, name: str, stance: str):
44        super().__init__(
45            name,
46            f"You are {name}. You firmly believe: {stance}. "
47            "State your position in 3 sharp sentences. No hedging.",
48        )

52class Critic(BaseAgent):
53    def __init__(self, name: str):
54        super().__init__(
55            name,
56            f"You are {name}, a sharp cross-examiner. "
57            "You will be given an opponent's argument. "
58            "Identify its weakest point and attack it in 2 sentences.",
59        )

63class Judge(BaseAgent):
64    def __init__(self):
65        super().__init__(
66            "Judge",
67            "You are an impartial judge. "
68            "You will receive two positions and two critiques. "
69            "Weigh the arguments. Pick the stronger side. "
70            "State your verdict and reasoning in 3 sentences. "
71            "Do not sit on the fence.",
72        )

75# ── Stage 1 — Fan‑Out: Parallel Positions ────────────────────────────────
76async def opening_positions(topic: str, debater_a: Debater, debater_b: Debater) -> tuple[str, str]:
77    print(f"\n{'='*60}")
78    print(f"TOPIC: {topic}")
79    print(f"{'='*60}")
80    print("\n[Stage 1] Opening positions …\n")
81    async def speak(agent: Debater) -> str:
82        print(f"\n── {agent.name} ──")
83        return await agent.think(f"State your position on: {topic}")
84    position_a, position_b = await asyncio.gather(
85        speak(debater_a),
86        speak(debater_b),
87    )
88    return position_a, position_b

95# ── Stage 2 — Cross‑Critique: Each Attacks the Other ─────────────────────
96async def cross_critique(
97    debater_a: Debater, position_a: str,
98    debater_b: Debater, position_b: str,
99) -> tuple[str, str]:
100    print("\n[Stage 2] Cross-critique …\n")
101    critic_a = Critic(f"{debater_a.name} (cross-examining)")
102    critic_b = Critic(f"{debater_b.name} (cross-examining)")
103    async def attack(critic: Critic, opponent_name: str, opponent_position: str) -> str:
104        print(f"\n── {critic.name} ──")
105        return await critic.think(f"{opponent_name} argued: {opponent_position}")
106    critique_by_a, critique_by_b = await asyncio.gather(
107        attack(critic_a, debater_b.name, position_b),
108        attack(critic_b, debater_a.name, position_a),
109    )
110    return critique_by_a, critique_by_b

119# ── Stage 3 — Fan‑In: Judge Decides ───────────────────────────────────────
120async def verdict(
121    debater_a: Debater, position_a: str, critique_by_b: str,
122    debater_b: Debater, position_b: str, critique_by_a: str,
123) -> str:
124    print("\n[Stage 3] Verdict …\n")
125    judge = Judge()
126    prompt = (
127        f"{debater_a.name} argued:\n{position_a}\n\n"
128        f"{debater_b.name} cross-examined {debater_a.name}:\n{critique_by_b}\n\n"
129        f"{debater_b.name} argued:\n{position_b}\n\n"
130        f"{debater_a.name} cross-examined {debater_b.name}:\n{critique_by_a}\n\n"
131        "Render your verdict."
132    )
133    print("── Judge ──")
134    return await judge.think(prompt)

141# ── Main ────────────────────────────────────────────────────────────────────
142async def main():
143    topic = "AI will eliminate more jobs than it creates in the next decade"
144    debater_a = Debater("Arjun", "AI is a net job destroyer — history will not repeat itself this time")
145    debater_b = Debater("Meera", "AI is a net job creator — every technology wave has expanded human work")
146    position_a, position_b = await opening_positions(topic, debater_a, debater_b)
147    critique_by_a, critique_by_b = await cross_critique(debater_a, position_a, debater_b, position_b)
148    final_verdict = await verdict(debater_a, position_a, critique_by_b, debater_b, position_b, critique_by_a)
149    print("\n" + "="*60)
150    print("VERDICT")
151    print("="*60)
152    print(final_verdict)

170import nest_asyncio
171nest_asyncio.apply()
172if __name__ == "__main__":
173    asyncio.run(main())

Understanding the Debate Pattern

This example implements a three‑stage debate between two agents. Agents adopt different roles — debater, critic, and judge — to challenge each other and arrive at a verdict. By staging the interaction into opening statements, cross‑examination, and judgement, the conversation surfaces richer reasoning than a single‑turn exchange.

BaseAgent

Lines 7‑17 define a reusable BaseAgent class. It stores an agent’s name, a system prompt (sys_prompt), conversation memory, and a Gemini client configured with a thinking budget of zero (no internal chain of thought). The think() method (lines 18‑37) constructs a single message from the user, streams the model’s response to stdout, accumulates the text, and appends the interaction to the agent’s memory. Returning the response allows higher‑level functions to capture what the agent said.

Debater, Critic and Judge roles

Three subclasses implement specific roles:

  • Debater (lines 42‑48) accepts a stance and sets the system prompt so the model firmly advocates that position in exactly three sentences, avoiding hedging.
  • Critic (lines 52‑59) is a cross‑examiner. Its prompt instructs the model to identify the weakest point in an opponent’s argument and attack it in two sentences.
  • Judge (lines 63‑72) weighs both sides. Its prompt explains that it will receive two positions and two critiques, and must render a verdict in three sentences without sitting on the fence.

Stage 1 – Opening Positions

The opening_positions() coroutine (lines 76‑88) kicks off the debate. It prints the topic and then defines an inner speak() function that asks each debater to state their position on that topic. The two debaters speak concurrently via asyncio.gather(), illustrating the fan‑out pattern: multiple agents can think at the same time. The function returns a tuple of the two positions.

Stage 2 – Cross‑Critique

In cross_critique() (lines 96‑110), each debater creates a temporary Critic to attack their opponent’s opening statement. Two attack() coroutines run in parallel: debater A’s critic critiques debater B’s position, and vice versa. This stage demonstrates concurrent criticism and returns both critiques.

Stage 3 – Verdict

The verdict() function (lines 120‑134) lets a Judge evaluate everything that came before. It builds a long prompt containing both positions and both critiques, separated with clear labels. The judge is then asked to render a verdict. The returned string contains the judge’s final decision along with reasoning.

Main Function

Finally, main() (lines 142‑152) wires the stages together. It defines a topic and two debaters with opposing stances, gathers their opening positions, orchestrates the cross‑critique, asks for a verdict, and prints the final judgement. The nest_asyncio.apply() call (lines 170‑171) enables running asynchronous code in a Jupyter environment, and asyncio.run(main()) executes the debate when the script runs as the main module.
