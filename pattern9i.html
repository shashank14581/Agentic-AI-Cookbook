Starting the MCP Server

The script begins by configuring a FastMCP server on port 8011. The search_private_knowledge tool is registered via a decorator; it inspects a lowercase query and returns a canned answer for known patterns (owners, thresholds, Silver Gate, Blue Mango). If no match is found, it returns a default message. The server is started in a separate thread only if the port is free.

MCP Client

The MCPClient wraps the protocol’s HTTP client. When call_tool() is invoked, it creates a streaming session, initialises it, and then calls the specified tool with the provided arguments. The result is extracted from the response and returned as plain text.

Gemini Agent with Tools

The asynchronous BaseAgent defines a single tool using types.FunctionDeclaration. In think(), the agent asks Gemini to generate content; the configuration includes the system prompt and the list of tools. When Gemini decides to call a tool, the agent prints the chosen tool name and arguments, then delegates to the MCPClient to execute the call. Otherwise the model’s free‑form answer is printed.

MCP RAG Agent and Orchestration

MCPRAGAgent simply sets a system prompt reminding Gemini to use the MCP tool for private facts. In the main() function the server is started (if needed), a client and agent are created, and a list of questions is processed asynchronously. Each question is printed along with the agent’s response, which may or may not involve a tool call.
