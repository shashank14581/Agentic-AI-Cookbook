Pattern 2: Personality and Memory

This pattern introduces memory and personality to agents, enabling them to remember facts and speak with character.

Summary

The BaseAgent gains a facts store and a context builder. Before sending input to Gemini, the agent extracts sentences like “I am ...” or “I like ...” and stores them as long‑term facts. When generating a response, the agent constructs a prompt that includes both its facts and the last few interactions, giving the model a richer context. Specialized agents such as a human and a god demonstrate how different system prompts yield different personalities.


import os
from google import genai
from google.genai import types

class BaseAgent():
    def __init__(self, name, sys_prompt, model="gemini-2.5-flash-lite"):
        self.name = name
        self.sys_prompt = sys_prompt
        self.memory = []
        self.facts_store = []
        self.client = genai.Client(api_key=os.environ.get("GEMINI_API_KEY"))
        self.model = model

    def build_context(self, input_text):
        history = ""
        for inp, out in self.memory[-3:]:
            history += f"user: {inp}\nsystem: {out}\n"

        facts_text = "\n".join(self.facts_store)

        full_input = f"""
FACTS:
{facts_text}

HISTORY:
{history}

NEW MESSAGE:
{input_text}
"""
        return full_input

    def extract_facts(self, input_text):
        if "I am" in input_text or "I like" in input_text:
            if input_text not in self.facts_store:
                self.facts_store.append(input_text)

    def think(self, input_text):
        self.extract_facts(input_text)
        full_input = self.build_context(input_text)

        contents = [
            types.Content(
                role="user",
                parts=[types.Part.from_text(text=full_input)]
            )
        ]

        config = types.GenerateContentConfig(
            thinking_config=types.ThinkingConfig(thinking_budget=0),
            system_instruction=[types.Part.from_text(text=self.sys_prompt)]
        )

        response = ""
        for chunk in self.client.models.generate_content_stream(
            model=self.model,
            contents=contents,
            config=config
        ):
            if chunk.text:
                print(chunk.text, end="")
                response += chunk.text

        print()
        self.memory.append((input_text, response))
        return response

class Human(BaseAgent):
    def __init__(self, name):
        super().__init__(
            name,
            f"You are {name}, a human. Talk naturally and casually."
        )

class God(BaseAgent):
    def __init__(self, name):
        super().__init__(
            name,
            f"You are {name}, a Hindu God. Be divine, wise, slightly playful."
        )

def orchestrate(human, god, turns=5):
    message = "Hi Indra!"
    for _ in range(turns):
        print(f"\n{human.name}: ", end="")
        human_out = human.think(message)
        print(f"\n{god.name}: ", end="")
        god_out = god.think(human_out)
        message = god_out

if __name__ == "__main__":
    human = Human("Shashank")
    god = God("Indra")
    orchestrate(human, god)

Lines 1–3: Import os and the genai modules. These provide access to the Gemini client and type definitions.

Lines 5–13: Initialize the BaseAgent. In addition to the memory list, this pattern introduces a facts_store to persist facts extracted from the conversation. A client is created with the Gemini API key.

Lines 14–32: Define build_context(), which composes a prompt including the stored facts, the last three interaction pairs, and the new message. The triple‑quoted string clearly separates these sections.

Lines 34–38: Define extract_facts(). If the incoming message contains “I am” or “I like” and it has not been stored before, the sentence is appended to facts_store. This creates a long‑term memory of facts.

Lines 39–71: Override the think() method. The agent extracts facts, builds a context that includes history and facts, then sends it to Gemini. The response is printed and appended to memory.

Lines 74–87: Define the Human and God subclasses. Each passes a personalized system prompt to the parent initializer, giving distinct conversational styles.

Lines 90–100: Implement orchestrate(), alternating messages between the human and the god. It starts with “Hi Indra!” and updates the message at each turn with the response from the previous agent.

Lines 103–107: The entry point creates the two agents and launches a five‑turn conversation. Over the course of the dialogue, the agents remember facts about themselves and speak with their unique personalities.
