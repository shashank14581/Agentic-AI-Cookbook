OVERVIEW

What this pattern teaches

The goal of this pattern is to turn unstructured text into structured data that can be queried later. A KnowledgeStore is used to persist entity extractions, and a specialized agent handles the task.

By defining a specific list of entity types (people, companies, locations, etc.), the agent acts as a parser that ignores noise and focuses only on data points that build a cohesive picture of a business or user interaction history.

CODE

Line-by-line walkthrough

import os
import json
from google import genai
from google.genai import types

class KnowledgeStore:
    def __init__(self):
        self.entities = []

    def write(self, agent_name, extracted_entities):
        self.entities.append({
            "agent": agent_name,
            "entities": extracted_entities
        })

    def read(self):
        if not self.entities: return "No entities extracted yet."
        return json.dumps(self.entities, indent=2)

class BaseAgent:
    def __init__(self, name, sys_prompt, knowledge_store, model="gemini-2.5-flash-lite"):
        self.name = name
        self.sys_prompt = sys_prompt
        self.knowledge_store = knowledge_store
        self.client = genai.Client(api_key=os.environ.get("GEMINI_API_KEY"))
        self.model = model

    def think(self, input_text):
        existing_knowledge = self.knowledge_store.read()
        final_input = f"\nExisting Knowledge Store:\n{existing_knowledge}\n\nCurrent Text:\n{input_text}\n\nExtract useful entities and return only valid JSON."
        
        contents = [types.Content(role="user", parts=[types.Part.from_text(text=final_input)])]
        config = types.GenerateContentConfig(thinking_config=types.ThinkingConfig(thinking_budget=0), system_instruction=[types.Part.from_text(text=self.sys_prompt)])

        response = ""
        for chunk in self.client.models.generate_content_stream(model=self.model, contents=contents, config=config):
            if chunk.text: response += chunk.text
        
        try: extracted_entities = json.loads(response)
        except json.JSONDecodeError: extracted_entities = {"raw_response": response}

        self.knowledge_store.write(self.name, extracted_entities)
        return extracted_entities

class EntityExtractionAgent(BaseAgent):
    def __init__(self, name, knowledge_store):
        super().__init__(name, "You are an Entity Extraction Agent. Extract: people, companies, locations, products, issues, actions, dates, relationships. Return ONLY valid JSON.", knowledge_store)

def orchestrate(agent, documents):
    for i, doc in enumerate(documents, start=1):
        agent.think(doc)
    print(agent.knowledge_store.read())

if __name__ == "__main__":
    docs = [
        "Customer Shashank bought a drill from Home Depot in Bengaluru and complained about delayed delivery.",
        "Bruno visited the Koramangala store and purchased paint. The billing team helped him resolve a refund issue."
    ]
    orchestrate(EntityExtractionAgent("EntityBuilder", KnowledgeStore()), docs)
Mental Model

Lines 6-19: The KnowledgeStore acts as a cumulative database. Each time an agent processes a document, its findings are appended here in a structured format.

Lines 30-49: The core logic. think() reads the store, combines it with the new text, and asks Gemini for JSON. We then attempt to parse that JSON before saving it.

Lines 52-54: The EntityExtractionAgent has a very strict system prompt. It specifies exactly what keys to look for, ensuring consistency across different documents.

Lines 56-59: The orchestrator demonstrates Iterative Extraction. By processing documents one by one, the Knowledge Store grows and provides more context for subsequent extractions.

Result: Instead of separate notes, you end up with a centralized, machine-readable database of every entity and interaction mentioned in your data set.
