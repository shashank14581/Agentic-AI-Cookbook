Pattern 9c – Knowledge Extraction

In this pattern, an agent turns messy natural language into structured data. It reads each document, extracts entities like people, companies and dates, and stores them in a knowledge store for later analysis.

25class BaseAgent:

33    def think(self, input_text):

37        final_input = f"""
38Existing Knowledge Store:
39{existing_knowledge}
40
41Current Text:
42{input_text}
44Extract useful entities and return only valid JSON.
45"""
47        contents = [types.Content(role="user", parts=[types.Part.from_text(text=final_input)])]
54        config = types.GenerateContentConfig(
55            thinking_config=types.ThinkingConfig(thinking_budget=0),
56            system_instruction=[types.Part.from_text(text=self.sys_prompt)]
57        )
59        response = ""
61        for chunk in self.client.models.generate_content_stream(
62            model=self.model, contents=contents, config=config
63        ):
66            if chunk.text:
67                print(chunk.text, end="")
68                response += chunk.text
70        print()
72        try:
73            extracted_entities = json.loads(response)

79        self.knowledge_store.write(self.name, extracted_entities)

Extracting Structured Knowledge

The goal of this pattern is to turn unstructured text into structured data that can be queried later. A knowledge store is used to persist entity extractions, and a specialised agent handles the extraction task.

KnowledgeStore

The KnowledgeStore class (lines 8‑22) is a simple container that accumulates extracted entities. The write() method appends a record containing the agent’s name and the entities it extracted. The read() method returns a human‑readable JSON string of all records or a placeholder if none exist.

BaseAgent

The BaseAgent (lines 25‑81) orchestrates the extraction workflow. In think(), it reads the existing knowledge store to provide context (line 35) and constructs a final_input string (lines 37‑45) that includes existing knowledge and the current text. The prompt instructs the model to extract useful entities and to return only valid JSON.

The prompt is sent to Gemini (lines 61‑68), and the response text is collected. The code then attempts to parse the response into JSON (lines 72‑75); if parsing fails, it stores the raw response. In either case, the extracted (or raw) data is saved to the knowledge store via write() (line 79).

EntityExtractionAgent

This subclass (lines 84‑107) defines a system prompt that instructs Gemini to extract specific entity types, including people, companies, locations, products, issues, actions, dates and relationships. The prompt emphasises that only valid JSON should be returned, and it forbids explanations.

Orchestration

The orchestrate() function (lines 110‑119) loops over a list of documents. For each document, it prints a separator, invokes agent.think(), and moves on. After processing all documents, it prints the final contents of the knowledge store. In the main block, a list of example customer interactions is passed into this function to demonstrate how information from multiple sources accumulates.
