Pattern 9h – Multi‑Hop Chain Retrieval

An agent breaks down complex questions into several smaller retrieval hops. It then combines the evidence from all hops to produce an informed answer.

19 class RAGMemory:
20     def __init__(self, embedding_dim=32):
21       self.word_to_id = {"<UNK>": 0}
22       self.embedding_dim = embedding_dim
23       self.model = None
24       self.store = []
25 
26     def build_vocab(self, texts):
27       for text in texts:
28         for word in text.lower().split():
29           if word not in self.word_to_id:
30             self.word_to_id[word] = len(self.word_to_id)
31 
32     def encode_text(self, text):
33       return torch.tensor([
34         self.word_to_id.get(w, 0)
35         for w in text.lower().split()
36       ], dtype=torch.long)
37 
38     def build_index(self, knowledge_base):
39       self.build_vocab(knowledge_base)
40       self.model = SimpleEmbeddingModel(len(self.word_to_id),self.embedding_dim)
41       for i, text in enumerate(knowledge_base, start=1):
42         token_ids = self.encode_text(text)
43         with torch.no_grad():
44           embedding = self.model(token_ids)
45         self.store.append({"doc_id":f"doc_{i}","text":text,"embedding":embedding})
46         print(f"Indexed doc_{i}")
47 
48     def retrieve(self, query, top_k=3):
49       query_ids = self.encode_text(query)
50       with torch.no_grad():
51         query_embedding = self.model(query_ids)
52       results = []
53       for item in self.store:
54         score = F.cosine_similarity(query_embedding.unsqueeze(0),item["embedding"].unsqueeze(0)).item()
55         results.append((score, item))
56       results.sort(key=lambda x: x[0], reverse=True)
57       context = ""
58       for score, item in results[:top_k]:
59         context += f"""
60   Document: {item["doc_id"]}
61   Similarity: {round(score, 4)}
62   Text: {item["text"]}
63   """
64       return context

Query Planning

The QueryPlannerAgent prompts Gemini to decompose a complex question into sequential retrieval hops. It returns a JSON object containing an array of queries. Each hop is intended to uncover one missing fact.

Executing the Hops

The MultiHopRAGAgent reads the query plan, prints it for inspection, and loops through each hop. For every hop it retrieves the most similar documents from the RAG memory, prints the context, and appends it to a growing evidence string. If the planner fails to return valid JSON, the agent falls back to treating the entire question as a single hop.

Answering with Combined Evidence

After gathering evidence across all hops, the agent constructs a final prompt that instructs Gemini to combine the retrieved information when answering. It explicitly forbids inventing facts not present in the retrieval results.

Example

The sample questions require multi‑step reasoning. For example, to find who monitors the queue used at low confidence, the agent must first discover which queue is used (Silver Gate) and then find which team monitors that queue (DS Ops team). The chain of retrieval hops makes such multi‑step questions answerable.
