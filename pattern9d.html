OVERVIEW

What this pattern teaches

The Chunk and Embed pattern (Pattern 13) demonstrates the mechanical foundation of RAG. Instead of relying on external vector databases, we implement a Chunker to split text with overlap and a SimpleEmbeddingModel using PyTorch.

This pattern highlights how Chunking controls the memory boundaries of the system and Embedding captures the semantic meaning. Together, they allow an agent to retrieve relevant facts from a large document to answer questions without "guessing" or hallucinating.

import os, torch
import torch.nn as nn
import torch.nn.functional as F
from google import genai
from google.genai import types

class Chunker:
    def __init__(self, chunk_size=30, overlap=5):
        self.chunk_size, self.overlap = chunk_size, overlap
    def chunk_text(self, text):
        words = text.split(); chunks, start = [], 0
        while start < len(words):
            end = start + self.chunk_size
            chunks.append(" ".join(words[start:end]))
            start = end - self.overlap
            if start >= len(words): break
        return chunks

class SimpleEmbeddingModel(nn.Module):
    def __init__(self, vocab_size, embedding_dim=32):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embedding_dim)
    def forward(self, token_ids):
        return self.embedding(token_ids).mean(dim=0)

class VectorStore:
    def __init__(self): self.store = []
    def add(self, chunk_id, text, embedding):
        self.store.append({"chunk_id": chunk_id, "text": text, "embedding": embedding})
    def search(self, query_emb, top_k=2):
        scores = [(F.cosine_similarity(query_emb.unsqueeze(0), item["embedding"].unsqueeze(0)).item(), item) for item in self.store]
        return sorted(scores, key=lambda x: x[0], reverse=True)[:top_k]

class RAGMemory:
    def __init__(self):
        self.chunker = Chunker(); self.vector_store = VectorStore()
    def build_index(self, doc):
        chunks = self.chunker.chunk_text(doc)
        # Vocabulary and Model initialization follows...
        for i, chunk in enumerate(chunks, start=1):
            self.vector_store.add(f"chunk_{i}", chunk, embedding)

class RAGAgent(BaseAgent):
    def think(self, question):
        context = self.rag_memory.retrieve(question)
        prompt = f"Answer only from context:\n{context}\n\nQuestion: {question}"
        res = self.client.models.generate_content(model="gemini-2.5-flash", contents=prompt)
        return res.text
Mental Model

Lines 7-17: The Chunker breaks long text into smaller pieces. The overlap parameter ensures that context appearing at the end of one chunk isn't lost for the beginning of the next.

Lines 19-24: Our neural SimpleEmbeddingModel. It uses a PyTorch embedding layer to turn word IDs into vectors, then averages them to create a single mathematical "fingerprint" for the chunk.

Lines 26-33: The VectorStore uses Cosine Similarity. It calculates the mathematical "angle" between your question and every chunk to find the best match.

Lines 35-43: RAGMemory orchestrates the indexing process: chunking the doc, building a vocabulary, and storing the vectors.

Lines 45-51: The RAGAgent. It doesn't answer from its own internal knowledge. It retrieves the context first, then uses it to construct a grounded, non-hallucinated response.
