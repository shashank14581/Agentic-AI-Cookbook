Pattern 9d – Chunking & Embedding

Retrieval‑augmented generation starts with chunking a document into overlapping segments, embedding them into vectors, and storing those vectors. Queries are embedded and compared to stored embeddings to retrieve relevant context.

2import torch

10class Chunker:
11    def __init__(self, chunk_size=30, overlap=5):
12        self.chunk_size = chunk_size
13        self.overlap = overlap
15    def chunk_text(self, text):
16        words = text.split()
17        chunks = []
18        start = 0
20        while start < len(words):
21            end = start + self.chunk_size
22            chunks.append(" ".join(words[start:end]))
23            start = end - self.overlap
25            if start >= len(words):
26                break
28        return chunks

31class SimpleEmbeddingModel(nn.Module):
32    def __init__(self, vocab_size, embedding_dim=32):
33        super().__init__()
34        self.embedding = nn.Embedding(vocab_size, embedding_dim)
36    def forward(self, token_ids):
37        word_embeddings = self.embedding(token_ids)
38        chunk_embedding = word_embeddings.mean(dim=0)
39        return chunk_embedding

42class VectorStore:
43    def __init__(self):
44        self.store = []
46    def add(self, chunk_id, text, embedding):
47        self.store.append({"chunk_id": chunk_id, "text": text, "embedding": embedding})
53    def search(self, query_embedding, top_k=2):
54        scores = []
56        for item in self.store:
57            score = F.cosine_similarity(query_embedding.unsqueeze(0), item["embedding"].unsqueeze(0)).item()
62            scores.append((score, item))
64        scores = sorted(scores, key=lambda x: x[0], reverse=True)
65        return scores[:top_k]

68class RAGMemory:
69    def __init__(self, chunk_size=30, overlap=5, embedding_dim=32):
70        self.chunker = Chunker(chunk_size, overlap)
71        self.embedding_dim = embedding_dim
72        self.word_to_id = {"": 0}
73        self.model = None
74        self.vector_store = VectorStore()
76    def build_vocab(self, chunks):
77        for chunk in chunks:
78            for word in chunk.lower().split():
79                if word not in self.word_to_id:
80                    self.word_to_id[word] = len(self.word_to_id)
82    def encode_text(self, text):
83        token_ids = []
85        for word in text.lower().split():
86            token_ids.append(self.word_to_id.get(word, 0))
88        return torch.tensor(token_ids, dtype=torch.long)
90    def build_index(self, document):
91        chunks = self.chunker.chunk_text(document)
92        self.build_vocab(chunks)
94        self.model = SimpleEmbeddingModel(vocab_size=len(self.word_to_id), embedding_dim=self.embedding_dim)
99        for i, chunk in enumerate(chunks, start=1):
100            token_ids = self.encode_text(chunk)
102            with torch.no_grad():
103                embedding = self.model(token_ids)
105            self.vector_store.add(chunk_id=f"chunk_{i}", text=chunk, embedding=embedding)
111            print(f"Stored chunk_{i}")
113    def retrieve(self, query, top_k=2):
114        token_ids = self.encode_text(query)
116        with torch.no_grad():
117            query_embedding = self.model(token_ids)
119        results = self.vector_store.search(query_embedding, top_k)
121        context = ""
123        for score, item in results:
124            context += f"""
125Chunk: {item["chunk_id"]}
126Score: {round(score, 4)}
127Text: {item["text"]}
128"""
130        return context

Chunking and Embedding Explained

This code shows how to implement a simple retrieval system from scratch using PyTorch and a vector store. It breaks a document into overlapping chunks, converts each chunk into an embedding, and stores these embeddings in a vector database. Queries are embedded and the most similar chunks are retrieved to provide context for a Gemini agent.

The Chunker

The Chunker class (lines 10‑28) splits a text into fixed‑size segments with a specified overlap. The overlap ensures continuity between adjacent chunks, which helps preserve context across boundaries.

Embedding Model and Vector Store

SimpleEmbeddingModel (lines 31‑39) is a minimal neural network that maps token IDs to vectors and averages them to produce a single embedding per chunk. The VectorStore (lines 42‑65) acts as a basic vector database: it stores chunk IDs, texts and embeddings and implements a cosine similarity search to find the top matching chunks for a given query embedding.

RAGMemory

The RAGMemory class (lines 68‑130) orchestrates chunking, vocabulary building and embedding. It maintains a mapping from words to integer IDs (word_to_id), builds a PyTorch model on demand and populates the vector store. The build_index() method processes a document, chunking it, building a vocabulary and embedding each chunk. retrieve() takes a query string, embeds it, and retrieves the most similar chunks from the vector store, returning a formatted context string with chunk IDs, scores and text.

BaseAgent and RAGAgent

BaseAgent (lines 133‑183) asks the RAG memory to retrieve context for a question and constructs a prompt that includes the retrieved chunks and the user’s question. It tells the model to answer strictly from the retrieved context and to admit when insufficient information is available. RAGAgent (lines 186‑197) simply sets a helpful system prompt that reminds the model not to hallucinate.

Putting It All Together

In the __main__ block, a sample document explains the RAG pipeline itself. RAGMemory builds an index from this document. A RetrieverAgent is then asked three questions about why chunking is used, what happens during query time, and what embedding controls. For each question, the agent retrieves the top two relevant chunks and answers using only that context, demonstrating the essence of retrieval‑augmented generation.
